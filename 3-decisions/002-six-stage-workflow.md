# D002: AI驱动开发六阶段流程

## 决策

流程为六阶段：

```
0 → Spec(需求分析) → TDD → Spec(开发需求) → Code → Test
```

### 各阶段定义

| 阶段 | 名称 | 性质 |
|------|------|------|
| 1 | 0 | 起点 |
| 2 | Spec (需求分析) | 输入，明确要做什么 |
| 3 | TDD | **协同核心**，团队对齐的锚点 |
| 4 | Spec (开发需求) | 中间状态，开发需求拆解 |
| 5 | Code | 编码实现 |
| 6 | Test | 测试验证 |

### 关键区分

**两个 Spec 的区别**：
- **TDD 之前的 Spec** = 需求分析（面向业务）
- **TDD 之后的 Spec** = 开发需求拆解（面向实现）

**协同核心**：
- **TDD = 协同核心** — 团队需要对齐的关键节点
- **Spec = 中间状态** — 个人连续工作的产物，非协同关键点

## 背景

两份参考材料对流程有不同描述，需要统一理解。
