# 2026-04-08 讨论：AI驱动开发流程框架

## 背景

公司团队决定全面采用AI驱动开发，希望制定标准化的流程与框架，实现人与人、人与AI、AI与AI之间的交互过程。

## 已确定的四阶段流程

### 阶段一：0 → Spec

流程：
1. 使用者通过 Claude Code 整理需求文档
2. Claude Code 交给 Codex 进行审核
3. 审核结果返回给 Claude Code
4. Claude 基于 GPT 的结果判断：take / 部分take / 拒绝（需附理由）
5. 若部分拒绝，将不同意部分返回 GPT 再看一遍
6. Claude 接受（若仍有问题，需人介入审核）

### 阶段二：Spec → TDD（Tech Design Document）

流程：
1. 由专人在本地AI上从头到尾生成
2. 基于git实现协同，让团队成员对TDD达成一致
3. TDD达成一致后，开发方向基本达成一致

### 阶段三：TDD → Code

由 PPM AI Agent 统筹管理开发任务。

### 阶段四：Code → Test

由 PPM AI Agent 统筹管理，形成闭环。

## PPM AI Agent 的角色扩展

- **原定位**：统筹 TDD→Code→Test 阶段
- **扩展意向**：加入前两阶段（0→Spec、Spec→TDD）统计人的执行状态和计划
